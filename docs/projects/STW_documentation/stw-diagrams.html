§ 01 Problem Statement

Why a hardware watchdog alone is not enough.

%%{init: {"theme": "dark", "flowchart": {"curve": "stepBefore"}} }%% flowchart TD HW["Hardware Watchdog\n(board-level timer)"] SERV["Servicing context\nstill running"] HW -- "timer serviced - no reset" --> SERV SERV --> S1["Single-task starvation\nHigh-priority task spins;\nlow-priority task never scheduled"] SERV --> S2["Deadlock / priority inversion\nSubset of tasks stall on\nshared resource"] SERV --> S3["Silent logical hang\nTask stays ready but stuck\nin loop or unreachable event"] SERV --> S4["No diagnosability\nReset discards context;\nwhich subsystem failed is unknown"] S1 & S2 & S3 & S4 --> STW STW["Software Thread Watchdog\nMonitors each task individually\nfor forward progress"] STW --> D["CAN diagnostic frame\nnames the specific stalled task\nbefore system-level recovery"] STW --> HW2["Hardware watchdog\nremains last line of defense"]
§ 02 Design Evolution: Timestamp to Counter

Why the first approach was scrapped and what replaced it.

%%{init: {"theme": "dark", "flowchart": {"curve": "stepBefore"}} }%% flowchart LR subgraph V1["Approach 1 — Timestamps (discarded)"] direction TB T1["Task loop iteration"] T2["Read current wall-clock time\n(multi-step, shared source)"] T3["Enter critical section"] T4["Write timestamp\n(multi-byte value)"] T5["Exit critical section"] T1 --> T2 --> T3 --> T4 --> T5 P1["Problems"] P1 --> PA["Critical section in hot path\nscheduling jitter and contention"] P1 --> PB["False alarms from clock\nwraparound / tick resolution"] P1 --> PC["Shared time service coupling\nadds stack depth and reentrancy risk"] end subgraph V2["Approach 2 — Monotonic Counters (adopted)"] direction TB C1["Task loop iteration"] C2["task_counter[id]++\n(volatile 32-bit, lock-free)"] C1 --> C2 M1["Monitor (once per cycle)"] M2{"counter !=\nlast_snapshot?"} M3["Update snapshot\nReset miss count\nMark HEALTHY"] M4["Increment miss count"] M1 --> M2 M2 -- Yes --> M3 M2 -- No --> M4 R1["Benefits"] R1 --> RA["No lock in heartbeat path\naligned 32-bit write is atomic"] R1 --> RB["No shared clock — detects\nchange, not elapsed time"] R1 --> RC["One memory write per iteration\nno call into time service"] end V1 -- "replaced by" --> V2
